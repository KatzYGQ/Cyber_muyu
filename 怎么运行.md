# 赛博木鱼 — 怎么运行（一步步来）

下面每一步都是在**干什么**、**你要做什么**。

---

## 第一步：安装依赖

**是啥意思？**  
这个项目用到了别人写好的代码包（比如 Next.js、Tailwind）。「安装依赖」就是让电脑把这些包下载到项目里，这样才能运行。

**你要做什么？**

1. 按 `Win + R`，输入 `cmd`，回车，打开「命令提示符」。
2. 输入下面两行（可以复制粘贴），每行输完按回车：

   ```
   cd c:\Users\lol19\QC_MATH_172_Spring_2025\hw02\b5
   npm install
   ```

3. 等它跑完（可能一两分钟），出现类似 `added xxx packages` 就表示成功。

---

## 第二步：配置环境变量（填数据库的地址和钥匙）

**是啥意思？**  
你的「木鱼」页面要把点击次数存到 insforge 的数据库里。电脑需要知道：  
- 数据库在哪个网址（**NEXT_PUBLIC_SUPABASE_URL**）  
- 用哪把「钥匙」才能读写（**SUPABASE_SERVICE_ROLE_KEY** 或 anon key）  

这些不能写在代码里，所以要放在一个叫 `.env.local` 的文件里，只有你本机有，别人看不到。

**你要做什么？**

1. 在项目文件夹里找到 **`.env.local`** 这个文件（和 `package.json` 同一层）。
2. 用记事本或 Cursor 打开它，你会看到两行要填的：
   - `NEXT_PUBLIC_SUPABASE_URL=` 后面要填你的 **insforge 项目网址**
   - `SUPABASE_SERVICE_ROLE_KEY=` 后面要填你的 **服务端密钥**（或 anon key）

3. **这两个值从哪里找？**
   - 如果你是用 **insforge**：在 insforge 后台 / 项目设置里，一般会有「Project URL」和「API Keys」（service_role 或 anon）。
   - 如果你暂时没有：可以先把这两行空着或随便填，页面能打开，但「全球累计功德」可能不更新，等你有 insforge 的 URL 和 key 再填上保存即可。

4. 保存 `.env.local` 后关掉。

---

## 第三步：启动开发服务器

**是啥意思？**  
「开发服务器」就是在本机跑起来一个小网站，只有你自己在浏览器里能打开，用来边改代码边看效果。

**你要做什么？**

1. 还是在「命令提示符」里，确保当前目录是项目目录（如果关了，就再执行一次第一步里的 `cd` 那一行）。
2. 输入：

   ```
   npm run dev
   ```

3. 等几秒，看到类似 `Local: http://localhost:3000` 就表示已经启动。

---

## 第四步：在浏览器里打开

**是啥意思？**  
用浏览器打开本机这个「小网站」，就能看到木鱼页面。

**你要做什么？**

1. 打开 Chrome / Edge 等浏览器。
2. 在地址栏输入：**http://localhost:3000**
3. 回车，就能看到黑色的木鱼页面。点中间的「木鱼」试试。

---

## 小结（你要做的就三件事）

| 顺序 | 做什么           | 在哪里做           |
|------|------------------|--------------------|
| 1    | 运行 `npm install` | 命令提示符，在项目目录 |
| 2    | 编辑 `.env.local`，填 URL 和 key（没有可先空着） | 用记事本或 Cursor 打开项目里的 `.env.local` |
| 3    | 运行 `npm run dev`，然后浏览器打开 http://localhost:3000 | 命令提示符 + 浏览器 |

如果你卡在某一步（比如不知道 insforge 的 URL 和 key 在哪找），可以说一下你用的是哪个平台（比如 Cursor + insforge），我可以按那个平台写更具体的找法。
